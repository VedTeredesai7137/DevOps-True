1. [cite_start]Open the Amazon S3 console[cite: 51].
2. [cite_start]Choose **Create bucket**[cite: 52].
3. Under the **General configuration** section:
    * [cite_start]**Bucket name:** Enter a unique name for your bucket (e.g., `ved-s3-lambda-trigger-bucket` as seen in the guide's examples)[cite: 54, 66]. 
    * [cite_start]**AWS Region:** Choose a Region, such as `US West (N. California) us-west-1`[cite: 55]. [cite_start]**Crucial Note:** Remember this region, as you *must* create your Lambda function in the exact same Region later[cite: 55].
4. **Settings to leave as Default:** The document does not specify changing any advanced settings. Therefore, you should leave the following sections exactly as they are:
    * [cite_start]**Block Public Access settings for this bucket:** Leave the default ticks on to ensure your bucket remains private[cite: 58].

    * [cite_start]**Advanced settings:** Leave as default[cite: 61].
5. [cite_start]Scroll to the bottom and choose **Create bucket**[cite: 59].

### Phase 3: Create the Lambda Function using a Blueprint
AWS provides templates (blueprints) that already have the necessary permissions and base code for S3 triggers configured.

1. [cite_start]Open the **Functions** page on the AWS Lambda console[cite: 166].
2. [cite_start]Choose **Create function**[cite: 167].
3. [cite_start]At the top of the creation page, select the **Use a blueprint** option[cite: 168].
4. [cite_start]Under the Blueprints section, type `s3` into the search box[cite: 169].
5. [cite_start]From the filtered results, select the Python blueprint named **`s3-get-object-python`**[cite: 171]. 
6. [cite_start]Choose **Configure**[cite: 171].
7. Under the **Basic information** section:
    * [cite_start]**Function name:** Enter `my-s3-function`[cite: 174].
    * [cite_start]**Execution role:** Select **Create a new role from AWS policy templates**[cite: 175].
    * [cite_start]**Role name:** Enter `my-s3-function-role`[cite: 175]. [cite_start]*(This role automatically grants CloudWatch log permissions, similar to the `ved-cloudwatch` IAM role shown in the document [cite: 109, 150])*
8. Under the **S3 trigger** section:
    * [cite_start]**Bucket:** Select the S3 bucket you created in Phase 1[cite: 178].
    * Keep the default event type to trigger on all object creations. [cite_start]*Note: The console will automatically modify your resource-based policy to allow S3 to invoke this function[cite: 181].*
9. [cite_start]Choose **Create function** at the bottom[cite: 182].

### Phase 4: Configure the Python Code
[cite_start]To achieve the aim of logging "An image has been added"[cite: 2], you need to slightly modify the blueprint code. 

1. [cite_start]On your Lambda function's page, scroll down to the **Code source** section[cite: 189].
2. [cite_start]Replace the default blueprint code in `lambda_function.py` with the following code derived from the document's screenshots and core objective[cite: 2, 182, 255]:

```python
import json
import urllib.parse
import boto3

print('Loading function')

s3 = boto3.client('s3')

def lambda_handler(event, context):
    # Get the object from the event and show its content type
    bucket = event['Records'][0]['s3']['bucket']['name']
    key = urllib.parse.unquote_plus(event['Records'][0]['s3']['object']['key'], encoding='utf-8')
    try:
        response = s3.get_object(Bucket=bucket, Key=key)
        print("CONTENT TYPE: " + response['ContentType'])
        
        # This is the specific log required by the experiment
        print('An image has been added')
        
        return response['ContentType']
    except Exception as e:
        print(e)
        print('Error getting object {} from bucket {}. Make sure they exist and your bucket is in the same region.'.format(key, bucket))
        raise e
```
3. Click **Deploy** to save your changes.
